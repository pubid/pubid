= ISO PubID V1 to V2 Migration Guide

== Overview

PubID V2 introduces a completely redesigned architecture with clean separation of concerns, improved performance, and new features. This guide helps you migrate from V1 (`pubid-iso` gem) to V2 (`pubid_new/iso` implementation).

== Why Migrate?

**V2 Benefits:**

* **Clean Architecture:** Three-layer separation (Parser, Builder, Identifier)
* **URN Generation:** RFC 5141-compliant URN support
* **Harmonized Stage Codes:** Updated to ISO's current system
* **Better Performance:** 5-6x faster with parser instance memoization
* **Type Safety:** Lutaml::Model-based serialization
* **Production Ready:** 92.84% test pass rate (2,654/2,859 tests)

== Breaking Changes

=== Namespace Change

[cols="1,1",options="header"]
|===
|V1 |V2

|`Pubid::Iso`
|`PubidNew::Iso`

|`Pubid::Iso::Identifier`
|`PubidNew::Iso::Identifiers::*`
|===

[example]
====
[source,ruby]
----
# V1
require 'pubid-iso'
id = Pubid::Iso::Identifier.parse("ISO 8601:2019")

# V2
require 'pubid_new'
id = PubidNew::Iso.parse("ISO 8601:2019")
----
====

=== Component API Changes

**Publisher:**

[source,ruby]
----
# V1
id.publisher           # => "ISO"
id.copublisher         # => ["IEC"]

# V2
id.publisher.body      # => "ISO"
id.publisher.copublisher  # => ["IEC"]
id.publisher.to_s      # => "ISO/IEC"
----

**Type:**

[source,ruby]
----
# V1
id.type                # => "TR"

# V2
id.type.abbr           # => "TR"
id.type.type_code      # => "tr"
----

**Number/Part:**

[source,ruby]
----
# V1
id.number              # => "27001"
id.part                # => "1"

# V2
id.number.value        # => "27001"
id.part.value          # => "1"
----

**Date:**

[source,ruby]
----
# V1
id.year                # => 2019

# V2
id.date.year           # => 2019
----

**Language:**

[source,ruby]
----
# V1
id.language            # => ["E", "F"]

# V2
id.languages.map(&:original_code)  # => ["E", "F"]
id.languages.map(&:code)           # => ["en", "fr"]
----

=== Stage Code Changes

V2 uses updated harmonized stage codes per ISO's current system:

[cols="1,1,1",options="header"]
|===
|Stage |V1 Code |V2 Code

|NP (New Proposal)
|`stage-00.00`
|`stage-10.00`

|PRF (Proof)
|`stage-60.00`
|`stage-50.00`

|WD (Working Draft)
|`stage-20.00`
|`stage-20.20`
|===

[example]
====
[source,ruby]
----
# V1
id = Pubid::Iso::Identifier.parse("ISO/NP 27001")
id.to_urn  # => "urn:iso:std:iso:27001:stage-00.00"

# V2
id = PubidNew::Iso.parse("ISO/NP 27001")
id.to_urn  # => "urn:iso:std:iso:27001:stage-10.00"
----
====

This change aligns with ISO's harmonized stage code definitions.

=== Rendering Changes

**Guide formatting:**

[source,ruby]
----
# V1
"ISO/Guide 1"    # slash before Guide

# V2
"ISO Guide 1"    # space before Guide
"ISO/IEC Guide 1"  # slash only between copublishers
----

**TypedStage rendering:**

[source,ruby]
----
# V1
typed_stage.abbreviation  # => "Amd"

# V2
typed_stage.canonical_abbreviation  # => "Amd"
typed_stage.original_abbr          # => "AMD" (if input was uppercase)
----

=== Class Structure Changes

**V1 had flat structure:**

[source]
----
Pubid::Iso::Identifier::*
----

**V2 has organized hierarchy:**

[source]
----
PubidNew::Iso::Identifiers::
  ├── SingleIdentifier (base documents)
  │   ├── InternationalStandard
  │   ├── Guide
  │   ├── TechnicalReport
  │   └── ...
  └── SupplementIdentifier (amendments)
      ├── Amendment
      ├── Corrigendum
      └── ...
----

== New Features in V2

=== URN Generation

V2 provides RFC 5141-compliant URN generation:

[source,ruby]
----
id = PubidNew::Iso.parse("ISO/IEC 27001:2013")
id.to_urn
# => "urn:iso:std:iso-iec:27001"

# With stage
id = PubidNew::Iso.parse("ISO/CD 27001")
id.to_urn
# => "urn:iso:std:iso:27001:stage-30.00"

# Supplements
id = PubidNew::Iso.parse("ISO 8601:2019/Amd 1:2023")
id.to_urn
# => "urn:iso:std:iso:8601:amd:2023:v1"
----

See README.adoc section "URN generation" for complete documentation.

== Common Migration Issues

=== Issue 1: Undefined method `value`

**Problem:**
[source,ruby]
----
id.number  # => PubidNew::Iso::Components::Code object
id.number.inspect  # NoMethodError: undefined method `value`
----

**Solution:**
[source,ruby]
----
id.number.value  # Correct
----

=== Issue 2: Publisher not a string

**Problem:**
[source,ruby]
----
id.publisher == "ISO"  # => false (comparing object to string)
----

**Solution:**
[source,ruby]
----
id.publisher.body == "ISO"  # => true
# OR
id.publisher.to_s == "ISO"  # => true
----

=== Issue 3: Stage codes differ

**Problem:**
[source,ruby]
----
# V1 URN
"urn:iso:std:iso:27001:stage-00.00"

# V2 URN
"urn:iso:std:iso:27001:stage-10.00"
----

**Solution:** This is intentional. V2 uses correct harmonized codes. Update your expectations:

[source,ruby]
----
# V2 is correct per ISO harmonized stage codes
expect(id.to_urn).to include("stage-10.00")
----

=== Issue 4: Language codes format

**Problem:**
[source,ruby]
----
# V1
id.language  # => ["E"]

# V2
id.languages.map(&:code)  # => ["en"]
----

**Solution:** Use `original_code` for single-char codes:

[source,ruby]
----
id.languages.map(&:original_code)  # => ["E"]
id.languages.map(&:code)           # => ["en"]
----

== Performance Considerations

V2 is significantly faster:

* **Simple identifiers:** 0.20ms (5,000/sec) vs V1's 1.0ms
* **Complex identifiers:** 0.46ms (2,174/sec) vs V1's 2.3ms
* **Memory:** 720 KB growth per 20k parses vs V1's 3.5 MB

**Optimization tip:** Reuse parser instance:

[source,ruby]
----
# V2 automatically memoizes parser instance
# Just call parse multiple times
1000.times do
  PubidNew::Iso.parse("ISO 8601:2019")
end
----
